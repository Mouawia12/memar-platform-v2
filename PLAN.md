## قرار المعمارية (Architecture)

**الطبقة الموحّدة:** الواجهة (HTML/JS الحالية) → **Laravel REST API** → **MySQL**. (المتصفح لا يتصل بقاعدة البيانات مباشرة — كل شيء عبر API + JSON).

- [ ] **بناء باك اند جديد بـ Laravel** (Eloquent ORM، Migrations، Controllers، Form Requests).
- [ ] المصادقة عبر **Laravel Sanctum** (توكن API — يصلح لتطبيق الموبايل لاحقًا) + bcrypt.
- [ ] تخزين الملفات عبر **Laravel Storage** على قرص السيرفر (NAS).
- [ ] التواصل الفوري (دردشة/إشعارات لحظية) عبر **Laravel Reverb + Echo** (WebSockets).
- [ ] توليد PDF عبر حزمة Laravel (dompdf / Browsershot).
- [ ] المهام الثقيلة (توليد المستندات، الإشعارات المجدولة) عبر **Queues + Scheduler**.
- [ ] **إعادة ربط الواجهة الحالية** (`erp/erp_app.js` + `shared/*`) لتستدعي Laravel API بدل Supabase المباشر و`localStorage` الوهمي.
- [ ] **الحفاظ على تصميم الواجهة الحالي** الذي أعجب العميل (نعيد التوصيل لا الشكل).
- [ ] **إلغاء باك اند Node/Express** (`src/`, `server.js`, Socket.io) بعد نقل المنطق المفيد منه (RBAC، منطق الوحدات).
- [ ] (اختياري — يُناقَش) استخدام **Filament** كلوحة إدارة داخلية سريعة لبعض وحدات الـERP.
